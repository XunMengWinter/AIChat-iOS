# 伴语 App 人工 UI 冒烟测试清单

本文档用于上架前、TestFlight 前或重要改动后的人工主流程检查。目标是快速确认 App 核心页面和关键路径可用，不替代完整功能测试、兼容性测试或安全测试。

## 测试准备

- 测试设备：优先使用 iPhone 17 模拟器；上架前至少补一次真机验证。
- 系统版本：iOS 17 及以上。
- 网络环境：正常 Wi-Fi 或蜂窝网络；如条件允许，补测一次弱网或网络切换。
- 后端环境：确认登录、角色、聊天、历史记录、图片上传相关接口可访问。
- 测试账号：使用 App Store Connect Review Notes 中提供的审核账号，不在正式 App 页面中展示测试账号。
- App 状态：建议先卸载旧版本，重新安装本次构建，避免本地缓存影响判断。

## 通过标准

- App 启动、页面切换和核心操作过程中不崩溃、不白屏、不长时间卡死。
- 核心页面内容可见，按钮可点击，输入框可输入，错误提示可理解。
- 主流程可以从启动走到登录、首页、聊天、设置和退出登录。
- 网络请求失败时有基础错误提示，不应让用户停留在无反馈状态。
- 页面在键盘弹出、图片选择、流式回复期间没有明显布局错乱。

## 冒烟测试步骤

### 1. 启动与引导页

- [ ] 冷启动 App，确认无崩溃、无白屏。
- [ ] 首次进入时显示角色引导页。
- [ ] 角色数据加载成功；加载中和失败状态有合理展示。
- [ ] 角色卡片可以横向滑动。
- [ ] 切换角色后分页指示和选中状态正常变化。
- [ ] 点击“开始聊天”后进入下一步流程。

### 2. 登录流程

- [ ] 登录页标题、手机号输入框、验证码输入框、获取验证码按钮、登录按钮正常显示。
- [ ] 手机号输入框只能输入数字，并限制为合理长度。
- [ ] 验证码输入框只能输入数字，并限制为 4 位。
- [ ] 未输入完整手机号时，“获取验证码”不可用或有明确提示。
- [ ] 点击“获取验证码”后出现发送中状态，并进入倒计时。
- [ ] 输入正确手机号和验证码后可以登录成功。
- [ ] 登录失败时显示明确错误提示，不崩溃、不丢失必要输入状态。
- [ ] 用户协议和隐私政策入口可访问；如当前版本还未实现，应记录为上架前阻塞项。

### 3. 首页

- [ ] 登录后进入首页。
- [ ] 首页背景、问候语、推荐角色、所有角色列表正常显示。
- [ ] 推荐角色头像和背景图加载正常；加载失败时有占位。
- [ ] 点击“立即聊天”可以进入对应角色聊天页。
- [ ] 横向角色列表可以滑动。
- [ ] 点击任意角色卡片可以进入对应角色聊天页。
- [ ] “继续聊天”入口行为正常。
- [ ] “聊天记录”入口可以滚动到最近聊天区域。
- [ ] 设置按钮可点击并进入设置页。

### 4. 聊天页文字消息

- [ ] 聊天页顶部返回按钮、角色头像、昵称、简介和更多菜单正常显示。
- [ ] 首次进入时能加载历史消息；无历史时显示角色开场白。
- [ ] 输入框可以输入文字。
- [ ] 空文本时发送按钮不可用或不可发送。
- [ ] 输入文字后发送按钮可用。
- [ ] 发送一条普通文字消息后，用户消息立即出现在消息列表。
- [ ] AI 回复以流式方式逐步显示。
- [ ] 回复完成后发送状态恢复，可以继续发送下一条消息。
- [ ] 聊天过程中自动滚动到底部，最新消息不被输入栏遮挡。
- [ ] 点击快捷话题后，输入框内容更新并可发送。
- [ ] 网络失败或服务端错误时显示错误提示，并允许用户继续操作。

### 5. 聊天页图片消息

- [ ] 点击图片按钮可以打开系统图片选择器。
- [ ] 选择图片后回到聊天页，并显示图片预览。
- [ ] 可以取消已选择的图片。
- [ ] 只发送图片时可以成功发出。
- [ ] 文本 + 图片同时发送时可以成功发出。
- [ ] 发送图片时输入区状态合理，不重复发送。
- [ ] 图片过大或处理失败时有错误提示。
- [ ] 发送完成后草稿图片被清空。

### 6. 聊天记录与清空

- [ ] 退出聊天页再进入同一角色，历史消息可以正常加载。
- [ ] 进入不同角色聊天页时，消息不会串到错误角色。
- [ ] 点击更多菜单中的“清空聊天”，出现清空操作入口。
- [ ] 清空聊天成功后消息列表回到角色开场白或空会话状态。
- [ ] 清空失败时显示错误提示，原消息不应被错误清空。

### 7. 设置与账号

- [ ] 设置页可以从首页进入。
- [ ] 设置页返回按钮可以回到首页。
- [ ] 当前手机号以脱敏形式展示。
- [ ] 当前角色信息展示正常。
- [ ] 点击“退出登录”后出现确认弹窗。
- [ ] 取消退出后仍停留在当前账号状态。
- [ ] 确认退出后回到引导或登录流程。
- [ ] 账号删除 / 注销入口可访问；如当前版本还未实现，应记录为上架前阻塞项。

### 8. 本地化与显示名称

- [ ] 中文系统环境下，桌面显示名称为“伴语”。
- [ ] 英文系统环境下，桌面显示名称为 “EchoMate”。
- [ ] App 内主要中文文案无明显错别字、截断或重叠。
- [ ] App Store 截图中展示的名称、功能和实际 App 一致。

### 9. 布局与可访问性基础检查

- [ ] 默认字体大小下，主要页面没有文字重叠、按钮文本截断或内容溢出。
- [ ] 开启较大字体后，登录页、首页、聊天页、设置页仍可基本操作。
- [ ] 键盘弹出时输入框可见，发送按钮可点击。
- [ ] 深色模式下没有严重不可读内容；如未适配深色模式，应确认 App 是否强制或主要使用浅色设计。
- [ ] VoiceOver 打开后，主要按钮具备可理解的读屏名称。

### 10. 异常场景快速检查

- [ ] 断网后启动 App，页面有加载失败或重试提示。
- [ ] 断网后发送消息，显示错误提示，不丢失用户已输入内容。
- [ ] 后端返回未授权时，App 能回到登录流程。
- [ ] 快速连续点击发送、返回、清空等按钮时不崩溃。
- [ ] App 进入后台再回到前台，当前页面状态基本保持。
